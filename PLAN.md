# ArXiv论文检索系统 - 完整技术规划

## 项目概述
构建一个基于RAG的ArXiv论文检索问答系统，支持论文检索、摘要问答、相关论文推荐等功能。

## 技术架构

### 1. 数据获取层
**ArXiv API**：
- API端点：`http://export.arxiv.org/api/query`
- 访问方式：REST API，支持XML格式
- 国内访问：**可正常访问**，无需科学上网（API服务稳定，但网站可能较慢）
- 限制：每3秒最多1次请求，建议批量获取

**数据获取流程**：
```python
# 示例API调用
import requests
url = "http://export.arxiv.org/api/query?search_query=cat:cs.AI&start=0&max_results=10"
response = requests.get(url)
# 解析XML获取论文信息
```

### 2. 数据处理层
**分块策略**：
- 论文摘要：保持完整（通常200-300词）
- 全文内容：按段落分块，chunk_size=512 tokens，overlap=50 tokens
- 元数据：标题、作者、分类、发布日期、PDF链接

**文本清洗**：
- 去除LaTeX公式标记
- 处理特殊字符
- 统一编码格式

### 3. 向量化层
**嵌入模型选择**（按优先级）：
1. **BGE-large-zh-v1.5**（推荐）：中文支持好，开源免费
2. **text-embedding-3-small**（OpenAI）：效果好，需API费用
3. **Sentence-BERT**：轻量级，本地部署

**向量维度**：1024维（BGE-large）或1536维（OpenAI）

### 4. 向量存储层
**向量数据库选型**：
1. **ChromaDB**（推荐）：轻量级，Python原生，适合原型
2. **FAISS**：Facebook开源，高性能，适合大规模
3. **Milvus**：分布式，适合生产环境

**存储结构**：
```
{
  "id": "arxiv_id",
  "embedding": [0.1, 0.2, ...],
  "metadata": {
    "title": "论文标题",
    "authors": ["作者1", "作者2"],
    "abstract": "摘要内容",
    "categories": ["cs.AI", "cs.CL"],
    "published": "2024-01-01",
    "pdf_url": "http://arxiv.org/pdf/..."
  }
}
```

### 5. 检索层
**混合检索策略**：
1. **向量检索**：余弦相似度，top_k=10
2. **关键词检索**：BM25算法，补充向量检索
3. **重排序**：使用rerank模型（如bge-reranker）提升精度

**检索优化**：
- 查询扩展：用LLM扩展用户查询
- 过滤条件：按分类、时间范围过滤
- 多路召回：结合向量+关键词+元数据

### 6. 生成层
**LLM集成**：
- **小米mimo2.5**：通过API调用，需配置token和key
- **提示词设计**：
```
你是一个论文检索助手。根据以下检索结果回答用户问题。

检索结果：
{context}

用户问题：{question}

请基于检索结果回答，如果信息不足请说明。
```

**参数配置**：
- temperature=0.3（保持准确性）
- max_tokens=1024
- top_p=0.9

### 7. 应用层
**前端界面**：
- **Streamlit**（推荐）：快速开发，Python原生
- **Gradio**：更简洁，适合演示
- **Web界面**：Flask/FastAPI + HTML/CSS/JS

**功能模块**：
1. 论文搜索：关键词+分类筛选
2. 问答系统：基于论文内容的问答
3. 相关推荐：相似论文推荐
4. 论文详情：查看摘要、PDF链接

## 关键技术点详解

### 1. 分块策略优化
- **语义分块**：使用NLTK/spaCy按句子分块
- **滑动窗口**：确保上下文连续性
- **元数据保留**：每个chunk保留论文元数据

### 2. 检索质量提升
- **查询理解**：用LLM解析用户意图
- **多路召回**：向量+关键词+分类
- **结果融合**：RRF（Reciprocal Rank Fusion）算法

### 3. 评估指标
- **检索评估**：召回率@K、MAP、NDCG
- **生成评估**：忠实度、相关性、完整性
- **用户评估**：回答准确性、有用性

### 4. 性能优化
- **缓存机制**：热门查询缓存
- **批量处理**：向量化批处理
- **异步加载**：提升响应速度
