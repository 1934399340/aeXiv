# ArXiv论文检索系统 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
随着AI和机器学习领域的快速发展，学术论文数量急剧增长。研究人员和开发者需要高效地检索、理解和应用最新研究成果。传统关键词搜索难以满足语义理解需求，而基于RAG（检索增强生成）的智能问答系统可以提供更精准的论文检索和知识问答服务。

### 1.2 项目目标
构建一个基于RAG的ArXiv论文检索问答系统，实现：
- 智能论文检索：支持语义搜索和关键词搜索
- 论文问答：基于论文内容的智能问答
- 相关推荐：发现相关研究论文
- 知识整合：将分散的论文知识结构化

### 1.3 目标用户
1. **研究人员**：快速找到相关论文，了解研究进展
2. **开发者**：检索技术实现方案，获取代码参考
3. **学生**：学习前沿技术，完成课程作业
4. **产品经理**：了解技术趋势，评估技术可行性

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 论文检索
- **语义搜索**：基于论文内容的相似度检索
- **关键词搜索**：传统BM25关键词匹配
- **混合搜索**：结合语义和关键词搜索
- **高级筛选**：按分类、时间范围、作者筛选
- **搜索建议**：输入时提供搜索建议

#### 2.1.2 论文问答
- **摘要问答**：基于论文摘要的问答
- **细节问答**：针对论文具体细节的问答
- **对比问答**：多篇论文的对比分析
- **总结问答**：研究领域的总结性回答

#### 2.1.3 相关推荐
- **相似论文**：基于内容相似度推荐
- **引用推荐**：推荐被引次数高的相关论文
- **作者推荐**：推荐同一作者的其他论文
- **领域推荐**：推荐同一研究领域的论文

#### 2.1.4 论文详情
- **基本信息**：标题、作者、摘要、分类
- **元数据**：发布日期、PDF链接、引用次数
- **内容预览**：摘要高亮显示
- **相关论文**：展示推荐论文列表

### 2.2 辅助功能

#### 2.2.1 用户界面
- **响应式设计**：支持桌面和移动端
- **暗色模式**：护眼模式切换
- **多语言支持**：中英文界面
- **快捷键**：提高操作效率

#### 2.2.2 数据管理
- **收藏功能**：保存感兴趣的论文
- **历史记录**：查看搜索和问答历史
- **导出功能**：导出搜索结果和笔记
- **分享功能**：分享论文链接和问答结果

## 3. 技术规格

### 3.1 系统架构

#### 3.1.1 整体架构
```
┌─────────────────────────────────────────────────────┐
│                    用户界面层                        │
│  Streamlit/Gradio Web界面 + REST API                │
├─────────────────────────────────────────────────────┤
│                    应用逻辑层                        │
│  检索引擎 + 问答生成 + 推荐系统                      │
├─────────────────────────────────────────────────────┤
│                    数据处理层                        │
│  数据获取 + 文本清洗 + 向量化 + 索引构建              │
├─────────────────────────────────────────────────────┤
│                    数据存储层                        │
│  向量数据库 + 元数据库 + 缓存数据库                  │
└─────────────────────────────────────────────────────┘
```

#### 3.1.2 技术栈
- **前端**：Streamlit 1.31.0 / Gradio 4.15.0
- **后端**：Python 3.10+, FastAPI
- **向量数据库**：ChromaDB 0.4.22
- **嵌入模型**：BGE-large-zh-v1.5
- **LLM**：小米mimo2.5 API
- **RAG框架**：LangChain 0.1.4
- **数据处理**：Pandas, NumPy, Requests

### 3.2 数据规格

#### 3.2.1 数据来源
- **ArXiv API**：`http://export.arxiv.org/api/query`
- **数据格式**：XML响应，解析为JSON
- **更新频率**：每日增量更新
- **数据量**：初始10万篇论文，持续增长

#### 3.2.2 数据结构
```json
{
  "id": "arxiv_id",
  "title": "论文标题",
  "authors": ["作者1", "作者2"],
  "abstract": "摘要内容",
  "categories": ["cs.AI", "cs.CL"],
  "published": "2024-01-01",
  "updated": "2024-01-02",
  "pdf_url": "http://arxiv.org/pdf/...",
  "abs_url": "http://arxiv.org/abs/...",
  "comment": "备注信息",
  "journal_ref": "期刊引用"
}
```

#### 3.2.3 向量规格
- **嵌入维度**：1024维（BGE-large）
- **向量类型**：浮点数数组
- **相似度计算**：余弦相似度
- **索引类型**：HNSW（分层可导航小世界图）

### 3.3 接口规格

#### 3.3.1 ArXiv API接口
```python
# 请求格式
GET http://export.arxiv.org/api/query?
    search_query={查询条件}&
    start={起始位置}&
    max_results={返回数量}&
    sortBy={排序字段}&
    sortOrder={排序方向}

# 响应格式（XML）
<feed>
  <entry>
    <id>arxiv_id</id>
    <title>论文标题</title>
    <summary>摘要内容</summary>
    <author><name>作者姓名</name></author>
    <category term="cs.AI"/>
    <published>2024-01-01T00:00:00Z</published>
    <link href="http://arxiv.org/pdf/..." title="PDF"/>
  </entry>
</feed>
```

#### 3.3.2 内部API接口
```python
# 论文检索接口
POST /api/search
{
  "query": "搜索关键词",
  "filters": {
    "categories": ["cs.AI"],
    "date_range": ["2024-01-01", "2024-12-31"]
  },
  "top_k": 10
}

# 论文问答接口
POST /api/ask
{
  "question": "用户问题",
  "paper_ids": ["arxiv_id1", "arxiv_id2"],
  "context_type": "abstract"  # 或 "full_text"
}

# 相关推荐接口
GET /api/recommend/{paper_id}?top_k=5
```
